CORE CAPABILITIES

/sys/modules — What SchemaClient does for you

Encrypted Tunnels

Expose your local server (localhost:8000) to the world with a secure public URL. Each session gets a unique subdomain — no code uploaded, no data stored remotely.

API Traffic Monitor

A built-in HTTP proxy that intercepts every request and response flowing through your APIs. See method, path, status, duration, headers, and full body — all in real-time.

Schema Validation

Define request and response schemas, then validate live traffic against them. Catch malformed payloads, missing fields, and type mismatches before they hit production.

OpenAPI Import

Import your existing OpenAPI / Swagger specs (JSON & YAML). SchemaClient automatically converts them into validation schemas — zero manual configuration required.

Request Collections

Organize your API requests into named collections. Save endpoints, headers, auth configs, and body payloads — then replay them with a single click.

Traffic Audit Logs

Every proxied request is persisted with full audit trails — headers, bodies, status codes, and timing. Filter, search, and review traffic history for debugging and compliance.

HOW IT WORKS

/proc/pipeline — From localhost to production-ready

  • 01.
    Start Tunnel
    Point SchemaClient at your local server (e.g. http://127.0.0.1:8000) and connect to the gateway.
  • 02.
    Get a Public URL
    Receive a unique, encrypted subdomain like myapp.schemaclient.net — shareable with your team instantly.
  • 03.
    Monitor Traffic
    The built-in proxy captures every request/response — method, path, status, headers, and body content.
  • 04.
    Validate Against Schema
    Attach a schema to any monitored endpoint. Incoming and outgoing payloads are validated in real-time.
  • 05.
    Debug & Iterate
    Click any log entry to inspect full request and response details. Fix issues instantly in your local dev environment.
  • 06.
    Ship With Confidence
    With validated schemas and audited traffic, deploy to production knowing your API contracts are enforced.
~/quickstart.sh
#!/bin/bash

# 1. Start the tunnel to expose localhost
schemaclient tunnel \
  --gateway https://mysmeclabs.com \
  --target http://127.0.0.1:8000 \
  --id myapp

# 2. Start the API proxy monitor
schemaclient monitor \
  --port 9000 \
  --target http://127.0.0.1:8000 \
  --schema ./api-schema.json
