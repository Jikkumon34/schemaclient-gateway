Core capabilities

Designed for local-first API teams shipping from desktop environments.

High-throughput tunneling

Keep local APIs reachable through a stable public URL while the gateway and desktop relay stream requests over websocket sessions.

Strict schema guardrails

Validate inbound and outbound payloads against your schema rules so malformed data is blocked before it reaches downstream systems.

Live API monitor

Inspect method, path, headers, body size, and response timing in real time to debug behavior and track access with full traceability.

Connection workflow

  1. 01.Create tunnel credentials from the gateway.
  2. 02.Connect desktop app with `tunnel_id` and `connect_key`.
  3. 03.Forward public traffic to localhost over websocket relay.
  4. 04.Apply schema validation and capture monitor events.
  5. 05.Return response to caller with end-to-end observability.

Built for the SchemaClient desktop app
